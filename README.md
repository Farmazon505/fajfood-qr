# QR на стол

Гостевой QR-сервис: вызов официанта с причиной, Telegram-уведомления, админ-панель, печать QR-кодов и карта лояльности, синхронизированная с CRM и iiko.

## Запуск

```bash
npm install
copy .env.example .env
npm run dev
```

Гостевая страница: `http://localhost:5174/t/table-1`

Админка: `http://localhost:5174/admin`

Для входа используются отдельные учетные записи администратора и владельца:

```env
ADMIN_USERNAME=admin
ADMIN_PASSWORD=replace-with-long-admin-password
OWNER_USERNAME=owner
OWNER_PASSWORD=replace-with-different-long-owner-password
```

В production пароли должны отличаться. Владелец видит закрытую оценку администраторов и может управлять должностями.

## Telegram

1. Создайте бота через BotFather.
2. Запишите токен в `TELEGRAM_BOT_TOKEN`.
3. Для локальной разработки поставьте `TELEGRAM_ENABLE_POLLING=true`.
4. В админке добавьте официанта и укажите его `telegramChatId`.

Если токен не задан, вызовы сохраняются в системе, но уведомления только логируются в консоль.

### Смена официанта

1. Официант нажимает в боте `Начать смену` и выбирает этаж или все этажи.
2. Система назначает его на столы выбранных зон и выдает чек-лист.
3. До выполнения обязательных пунктов уведомления от столов заблокированы.
4. После допуска бот доставляет ожидающие вызовы и все новые обращения.
5. Кнопка `Закончить смену` снимает официанта со столов, удаляет его активные вызовы из чата и отключает уведомления.

Состав чек-листа, обязательность пунктов, история смен, замечания и рейтинг настраиваются во вкладках `Чек-листы` и `Смены и рейтинг`. Каждый пункт привязан к должности сотрудника. Выполненный пункт по умолчанию дает 5 звезд, администратор может поставить от 1 до 5 звезд, невыполненный пункт дает 0. Пункты с выключенным флажком `Учитывать в рейтинге` не входят в расчет.

Рейтинг смены равен средней оценке всех учитываемых пунктов. Личный рейтинг сотрудника — среднее по оцененным сменам; в таблице дополнительно показаны накопленные звезды, выполнение задач и динамика последних смен. Сравнение и утреннее Telegram-сообщение строятся отдельно внутри каждой должности.

### ИИ-анализ эффективности

OpenRouter анализирует массовые и индивидуальные паттерны выполнения задач. Ключ используется только сервером:

```env
AI_BASE_URL=https://openrouter.ai/api/v1
AI_PROXY_URL=
OPENROUTER_API_KEY=replace-with-openrouter-key
AI_MODEL=openai/gpt-4o-mini
AI_REQUEST_TIMEOUT_MS=12000
```

Если OpenRouter недоступен, интерфейс автоматически показывает локальный анализ без отправки данных во внешний API.

Если назначенный официант не завершил обязательный чек-лист, вызов получает администратор своей зоны. Через четыре минуты бот предупреждает администратора, а через пять минут без принятия отправляет вызов владельцу.

Повторные обращения одного стола не создают новые сообщения. Бот редактирует первое сообщение и увеличивает счетчик. После `Принято` счетчик следующего обращения начинается с единицы, а `Готово` закрывает поток и удаляет сообщение из чатов назначенных официантов.

## Лояльность

`Qrnastol` не хранит ключ iiko. Регистрация отправляется по серверному каналу в CRM:

```env
CRM_BASE_URL=http://127.0.0.1:3000
CRM_LOYALTY_SERVICE_SECRET=shared-random-secret-with-at-least-32-characters
```

В CRM должен быть тот же `CRM_LOYALTY_SERVICE_SECRET`, а также:

```env
LOYALTY_WELCOME_BONUS_AMOUNT=500
IIKO_LOYALTY_WALLET_ID=
```

При регистрации система отдельно фиксирует обязательное согласие на обработку персональных данных и необязательное согласие на рекламные сообщения. Для доказательства сохраняются версия и SHA-256 текста, дата, IP и User-Agent.

Номер подтверждается бесплатно через контакт текущего пользователя в Telegram или MAX. Сайт не принимает номер из мессенджера на доверии: CRM проверяет Telegram `user_id` либо подпись контакта MAX, срок действия ссылки и совпадение номера с анкетой.

Для безопасного тестового запуска укажите разрешенные номера через запятую:

```env
LOYALTY_REGISTRATION_ALLOWLIST=+79990000000,+79990000001
```

## Что уже есть

- QR-страница столика `/t/:slug`.
- Настраиваемые кнопки вызова: счет, заказ, помощь, кальянщик и любые другие.
- Telegram-смены, обязательный чек-лист, агрегированные вызовы с кнопками "Принято" и "Готово".
- Админка для заведения, акций, кнопок, столов, официантов и заявок лояльности.
- Печатный лист QR-кодов с короткой инструкцией для гостей.
